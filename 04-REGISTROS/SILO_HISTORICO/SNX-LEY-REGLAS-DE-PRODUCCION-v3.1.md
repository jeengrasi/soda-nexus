
SNX-LEY-REGLAS-DE-PRODUCCION  
Versión 3.1 — Estado Soberano

1. Propósito
Esta ley define cómo deben generarse, entregarse y guardarse todos los documentos y scripts del proyecto SODA-NEXUS. Su objetivo es eliminar errores, evitar amnesia, mantener orden y garantizar continuidad operativa.

2. Regla del Documento en Texto Plano
Todos los documentos deben ser entregados en texto plano. No se permiten bloques de código, backticks, triple comillas ni símbolos que puedan ser interpretados como comandos por Termux.

3. Regla del Documento Listo para Guardar
Cada documento debe guardarse usando cat con here-doc. El documento debe venir completo y no debe requerir edición manual dentro del EOF.

4. Regla del Puente de Comunicación
El usuario es el puente entre IAs y Termux. Cada documento debe incluir instrucciones claras, pasos exactos, observaciones y pie institucional.

5. Regla del Documento Autocontenido
Cada documento debe incluir contexto, propósito, contenido principal, observaciones y pie institucional. Nada puede depender de memoria previa.

6. Regla del Script con Observaciones
Todo script debe incluir propósito, riesgos, dependencias, advertencias, notas de auditoría y pie institucional.

7. Regla del Chat Único
La mayor parte del trabajo debe hacerse en un solo chat para evitar pérdida de contexto.

8. Regla de la Estructura Soberana
El sistema solo puede tener seis carpetas: 00-GOBIERNO, 01-MEMORIA, 02-SISTEMA, 03-OPERACIONES, 04-REGISTROS, 05-DOCUMENTACION.

9. Regla de No Dependencia de Terceros
El proyecto no puede depender de ventas, servicios, freelancing ni clientes. Solo se permiten rutas autónomas de monetización digital.

10. Regla del Monitor Central
El monitor es el centro operativo del sistema. Debe ser accesible, modular, seguro y resistente a amnesia.

11. Regla de Antiamnesia
El sistema debe estar protegido mediante estructura fija, documentación obligatoria, scripts con observaciones, estado único y auditorías periódicas.

12. Regla de Producción Continua
Cada avance debe documentarse, archivarse, ejecutarse, verificarse y auditarse.
