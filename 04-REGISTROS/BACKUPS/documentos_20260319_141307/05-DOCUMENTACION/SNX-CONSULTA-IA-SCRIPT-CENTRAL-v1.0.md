SNX-CONSULTA-IA-SCRIPT-CENTRAL  
Versión 1.0 — Documento de consulta para múltiples IAs

Este documento solicita investigación, análisis y propuestas técnicas sobre la arquitectura del IA-SCRIPT CENTRAL y los IA-scripts derivados del proyecto SODA-NEXUS. La consulta debe ser respondida con investigación real, referencias, fuentes, casos de uso y casos de éxito. La respuesta debe ser global, abierta y sin sesgos.

1. Contexto del proyecto
SODA-NEXUS es un sistema autónomo diseñado para generar ingresos digitales sin depender de ventas, servicios, freelancing ni clientes. Todo el sistema vive en Termux, dentro de una estructura soberana de seis carpetas: 00-GOBIERNO, 01-MEMORIA, 02-SISTEMA, 03-OPERACIONES, 04-REGISTROS y 05-DOCUMENTACION. El sistema debe ser ordenado, auditable, resistente a amnesia y capaz de mantener su historia y contexto sin depender de la memoria de una IA específica.

2. Problema a resolver
El proyecto necesita un IA-SCRIPT CENTRAL que actúe como guardián del sistema. Este script debe proteger la estructura, la lógica, la memoria, la historia, la seguridad y la coherencia del proyecto. Además, debe coordinar IA-scripts derivados que auditen cada carpeta soberana. El objetivo es evitar desorden, pérdida de contexto, corrupción de archivos, creación de carpetas no autorizadas y cualquier intento fraudulento.

3. Funciones esperadas del IA-SCRIPT CENTRAL
Debe verificar que solo existan las seis carpetas soberanas.
Debe detectar carpetas o archivos no autorizados.
Debe auditar la historia y la memoria institucional.
Debe verificar que las leyes, el contexto y las decisiones existan y estén completas.
Debe coordinar IA-scripts derivados para auditar cada carpeta.
Debe generar reportes y alertas.
Debe proteger contra intentos fraudulentos o anómalos.
Debe mantener el contexto del proyecto para cualquier IA o persona que consulte el sistema.
Debe actuar como sistema inmune, firewall institucional y auditor total.

4. Funciones esperadas de los IA-scripts derivados
Cada carpeta soberana debe tener un IA-script local que:
Conozca qué archivos deben existir.
Conozca qué archivos no deben existir.
Detecte anomalías internas.
Genere un reporte para el IA-SCRIPT CENTRAL.
No tome decisiones por sí mismo.

5. Lo que se solicita a las IAs
Se solicita investigación global, abierta y sin sesgos sobre:
Modelos de arquitectura para sistemas de auditoría autónoma.
Casos de éxito de sistemas de auto-verificación y auto-reparación.
Patrones de diseño para guardianes de integridad en sistemas distribuidos.
Mecanismos de antiamnesia y preservación de contexto.
Sistemas de seguridad que detectan y bloquean acciones no autorizadas.
Estrategias para mantener coherencia documental en proyectos de larga duración.
Diseños de agentes centrales que coordinan agentes derivados.
Buenas prácticas para auditorías automáticas en entornos Unix-like.
Modelos de reporte estructurado para monitores y paneles de control.

6. Requisitos de la respuesta
La respuesta debe incluir:
Fuentes verificables.
Referencias técnicas.
Casos de uso reales.
Casos de éxito documentados.
Propuestas de arquitectura.
Alternativas y comparaciones.
Riesgos y consideraciones.
Sugerencias de implementación.
Recomendaciones globales sin sesgos.

7. Objetivo final de la consulta
Obtener una visión amplia, técnica y fundamentada que permita diseñar:
El IA-SCRIPT CENTRAL.
Los IA-scripts derivados.
El sistema de auditoría.
El sistema de antiamnesia.
El sistema de seguridad.
El sistema de preservación de contexto.
El sistema de reportes para el monitor.
